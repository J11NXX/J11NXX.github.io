# Как развернуть сайты — памятка для Узбекистана 🇺🇿

У вас три статических сайта (только HTML/CSS/JS, без сервера и базы).
Это самый простой случай: **бесплатный хостинг + быстрый деплой**.

Структура папки:
```
sites/
├── obzhig/index.html   → интернет-магазин кофе
├── forma/index.html    → визитка арх-бюро
└── ritm/index.html     → лендинг приложения
```
Каждый сайт лежит в своей папке как `index.html` — так платформы открывают его автоматически.

---

## Главное про Узбекистан в двух словах

| Вопрос | Ответ |
|---|---|
| GitHub Pages / Netlify / Vercel доступны из УЗ? | **Да**, работают без VPN |
| Нужна ли карта для бесплатного хостинга? | **Нет** — бесплатных тарифов хватит |
| Чем платить за зарубежные сервисы (если позже понадобится)? | Нужна **Visa/Mastercard** карта (Uzcard/Humo за рубежом не работают) |
| Где брать .uz домен | У **местного аккредитованного регистратора**, оплата в сумах через Payme/Uzcard/Humo |

## Вариант 1. GitHub Pages — рекомендую для старта

Лучший выбор для фрилансера: сайты в сети **и** портфолио кода, плюс осваиваете Git.
Полностью бесплатно, карта не нужна.

1. Зарегистрируйтесь на **github.com** (бесплатно, работает из УЗ).
2. Создайте репозиторий, например `portfolio`.
3. Загрузите содержимое папки `sites` (кнопка **Add file → Upload files**, перетащите папки).
4. **Settings → Pages** → в разделе *Build and deployment* выберите ветку `main`, папку `/ (root)` → **Save**.
5. Через 1–2 минуты сайты будут по адресам:
   - `https://ВАШ-ЛОГИН.github.io/portfolio/obzhig/`
   - `https://ВАШ-ЛОГИН.github.io/portfolio/forma/`
   - `https://ВАШ-ЛОГИН.github.io/portfolio/ritm/`

> Совет: каждую работу можно вынести в отдельный репозиторий — тогда ссылки будут короче и аккуратнее для клиента.

## Вариант 2. Netlify Drop — за 2 минуты, без кода

Самый быстрый способ показать результат заказчику.

1. Откройте **app.netlify.com/drop**
2. Перетащите туда папку **одного** сайта (например, `obzhig`).
3. Сразу получаете живую ссылку вида `random-name.netlify.app`.
4. Повторите для `forma` и `ritm` — будет три ссылки.

Бесплатно, карта не нужна. Позже можно завести аккаунт, чтобы ссылки не пропадали и можно было менять имя.

## Вариант 3. Vercel

Похож на Netlify (перетаскивание или подключение GitHub). Бесплатный тариф, карта не нужна.
Берите его, если в будущем перейдёте на React / Next.js.

## Свой домен

### Если хотите красивый `.uz` адрес (например `obzhig.uz`)
`.uz` регистрируют только **местные аккредитованные регистраторы**, оплата в сумах:
- **ahost.uz** — один из самых дешёвых, от ~27 000 сум/год
- **hostmaster.uz**, **itsg.uz**, **eskiz.uz**, **airnet.uz** — тоже официальные

Нюансы:
- Оплата через **Payme / Uzcard / Humo / Visa** — местная карта подходит.
- Открытая зона `.uz` — без ограничений. А вот `.com.uz`, `.co.uz`, `.net.uz`, `.org.uz` доступны **только гражданам Узбекистана**.
- Активация ручная — занимает от нескольких минут до нескольких часов (в выходные не активируют).
- После покупки в настройках домена пропишите DNS, которые даст Netlify/Vercel (или включите домен в GitHub Pages → Settings → Pages → Custom domain).

### Если хотите `.com` (например через Namecheap, GoDaddy)
Здесь нужна **международная карта Visa/Mastercard**, потому что Uzcard/Humo за границей не принимаются. Варианты:
- Виртуальная **Visa** в приложении банка: NBU (*Milliy*), Aloqabank (*Zoomrad*), **Uzum Bank** и др.
- Либо **ко-бейдж карта Visa-Humo / Mastercard-Uzcard** (выпускают многие банки, ~30 000 сум).
Пополняете такую карту в сумах — оплата конвертируется автоматически.

## Локальный `.uz`-хостинг (если клиенту нужен сайт «у нас»)

Иногда заказчик в УЗ хочет, чтобы сайт физически хостился в стране (быстрее открывается локально, оплата в сумах, договор на местную компанию). Тогда вместо Netlify берёте хостинг у **ahost.uz / hostmaster.uz / itsg.uz**: покупаете тариф, заходите в панель (обычно cPanel) и загружаете файлы в папку `public_html`. Оплата — Payme/Uzcard/Humo в сумах.

## Что я бы сделал на вашем месте

1. Сейчас: выложите всё на **GitHub Pages** — бесплатно, без карты, заодно навык Git в резюме.
2. Когда появятся первые клиенты: оформите **виртуальную Visa** в банке — пригодится для доменов и платных сервисов.
3. Под реального заказчика в УЗ: `.uz`-домен у местного регистратора + при необходимости локальный хостинг.
